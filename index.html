關於我（About 區占位內容）

這裡目前是 About 區的占位內容，之後會用 Section SDD 的 Prompt 產生更完整的自我介紹，例如學習背景、前端技術經歷、職涯目標與個人風格描述等。

開發者在之後替換時，請將新的 section 程式碼（包含標題、段落、清單、圖片等所有元素）整段貼在 SECTION:ABOUT-START 與 SECTION:ABOUT-END 這兩個註解中間，完全取代現在這一區，不需要保留目前的文字。

建議未來可以在這一區加入 1 張個人頭像或工作情境圖片，以及 1～3 個小重點條列，讓訪客快速理解你是怎樣的前端工程師。

技能（Skills 區占位內容）

這裡目前是 Skills 區的占位內容，未來會使用 Section SDD 的 Prompt 產生包含前端技術、工具、流程相關的完整技能列表與視覺化呈現（例如標籤、長條圖或分組清單）。

  • 可在這裡放入 HTML、CSS、JavaScript、框架與工具等技能分類。
  • 亦可加入「熟悉程度」、「使用情境」、「實戰經驗」等輔助說明。

之後請用 Section SDD 的 Prompt 生成新的 <section id="skills"> 區塊，並將產出的 section 程式碼整段貼在 SECTION:SKILLS-START 與 SECTION:SKILLS-END 這兩個註解之間，完全覆蓋目前這一區的內容。

作品集（Projects 區占位內容）

這裡目前是 Projects 區的占位內容，未來會用 Section SDD 的 Prompt 產生真正的作品集區塊，例如卡片式專案列表、縮圖、專案標題、簡短說明與連結按鈕等。

此處可作為「作品卡片」的占位示意，例如：個人作品標題、簡短功能說明、使用技術、前往 GitHub 或 Demo 的連結。
未來替換為實際專案資料時，請在新的 section 中自行定義 class 與排版，整段貼在兩個註解中間即可，不需要與現在的 placeholder 結構相同。

之後請用 Section SDD 的 Prompt 產生新的 Projects 區 section，再將生成的 section 程式碼整段貼到 SECTION:PROJECTS-START 和 SECTION:PROJECTS-END 之間，完整取代目前的占位內容。

聯絡我（Contact 區占位內容）

這裡目前是 Contact 區的占位內容，未來會使用 Section SDD 的 Prompt 設計正式的聯絡區，包括表單、社群連結、Email 與其他聯絡說明等。

  • 電子郵件（示意）： hello@example.com
  • 手機號碼（示意）： +886 900 000 000
  • GitHub（示意）： https://github.com/yourname
  • LinkedIn（示意）： https://www.linkedin.com/in/yourname

之後若使用 Section SDD 產生新的 Contact 區設計，請將包含所有聯絡方式、說明文字與可能的表單 HTML 的 section 程式碼整段貼在 SECTION:CONTACT-START 與 SECTION:CONTACT-END 之間，完全取代目前這一區的占位內容。
